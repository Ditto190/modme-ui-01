## Project Notes

Built the Universal Chat Ingestion system for Dylan's Phoenix observability pipeline. Key architectural decision: AI builds the tools offline, tools run the pipeline at runtime (no LLM calls in production). The system has 3 layers:

1. TypeScript type system (agent-generator/src/chat-formats/) - ChatFormatDescriptor with Zod fingerprints, FieldMappings, normalizer with pluggable extractors. Pattern mirrors existing schema-crawler.ts → registry-fetcher.ts relationship.

2. Python bridge extended (agent/observability/trace_bridge_api.py v2.0) - New /ingest endpoint accepts UniversalTurnPayload (format-agnostic). Creates AGENT→LLM→TOOL span hierarchy using OTel SDK + OpenInference conventions. The existing /upload stays for backward compat.

3. n8n workflow (ID: dfHBSbrEHbUi4H8B) - 7 nodes: Webhook → Detect (fingerprint registry in Code node) → If → Normalize (format-specific extractors in Code node) → HTTP to bridge /ingest → Response. Pure JavaScript, no LLM calls.

Key insight: Zod schema IS the fingerprint. safeParse() simultaneously detects AND validates. schema-crawler can generate these from JSON Schema.

Files created:
- agent-generator/src/chat-formats/types.ts (all Zod schemas)
- agent-generator/src/chat-formats/fingerprint.ts (detection engine)
- agent-generator/src/chat-formats/normalizer.ts (extraction engine)
- agent-generator/src/chat-formats/registry.ts (format registry)
- agent-generator/src/chat-formats/formats/copilot-chat.ts (first adapter)
- agent-generator/src/chat-formats/index.ts (barrel export)

Bridge changes: trace_bridge_api.py got /ingest endpoint + /formats info endpoint + _upload_universal_turn() function.
