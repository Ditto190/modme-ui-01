# Handoff Report: Milestone 2 (Knowledge/Memory Scaffold)

## 1. Observation
- The `.lean-ctx/` directory currently does not exist in the repository root.
- The `scripts/` directory exists, but `scripts/lean-ctx-index.ps1` does not.
- `PROJECT.md` identifies M2 (Knowledge/Memory Scaffold) as "PLANNED".
- `design.md` (Section 2) details the exact directory structure required (`.lean-ctx/memory/knowledge`, `.lean-ctx/memory/sessions`, `.lean-ctx/memory/diary`).
- `design.md` specifies 4 actions for `scripts/lean-ctx-index.ps1`:
  1. Aggregate JSON lines from `diary/*.md` and `sessions/*.md`.
  2. Extract findings with `confidence >= 0.9`.
  3. Call `lean-ctx ctx_knowledge consolidate`.
  4. Alert if architectural decisions need manual promotion to `AGENTS.md`.
- Side note: The `.lean-ctx.toml` configuration from Milestone 1 is missing several properties specified in `design.md` (e.g., `graph_index_max_files = 15000`, `memory_profile = "adaptive"`, `memory_cleanup = "shared"`, `multi_agent_sync = true`).

## 2. Logic Chain
- To fulfill M2, the worker must first scaffold the missing directories so that the memory profiles defined in the `.lean-ctx.toml` (which should be adaptive/shared) have a valid place to store data.
- The `scripts/lean-ctx-index.ps1` must be created. Since it must parse "JSON lines" from `.md` files, it will need to read each line, attempt to parse it using `ConvertFrom-Json`, filter by the `confidence` property, execute the consolidation command, and finally check properties (like `type` or `tags` or textual matches for "architecture" or "decision") to print a warning.
- Although M1 is marked as "DONE", the missing properties in `.lean-ctx.toml` could cause the M2 memory structure to be unused if `adaptive` memory is not enabled. The implementer should ensure these are added as a prerequisite.

## 3. Caveats
- The exact format of the JSON lines generated by the agents in `diary/` and `sessions/` is not strictly defined in `design.md` beyond having a `confidence` field and likely a `type`, `tags`, or `content` field. The PowerShell script will need a `try/catch` around JSON parsing to safely ignore regular markdown text.
- The exact CLI arguments for `lean-ctx ctx_knowledge consolidate` are assumed to not require piping the filtered JSON directly, as the design simply says "Calls `lean-ctx ctx_knowledge consolidate` to merge these findings".
- M1 completeness was not the primary focus, but missing `.lean-ctx.toml` entries were noticed and should be addressed for M2 to work properly.

## 4. Conclusion
The worker should implement M2 with the following concrete step-by-step strategy:
1. Update `.lean-ctx.toml` to append the missing settings from `design.md`:
   ```toml
   graph_index_max_files = 15000
   memory_profile = "adaptive"
   memory_cleanup = "shared"
   multi_agent_sync = true
   ```
2. Run `New-Item -Path ".lean-ctx/memory/knowledge", ".lean-ctx/memory/sessions", ".lean-ctx/memory/diary" -ItemType Directory -Force`.
3. Create `scripts/lean-ctx-index.ps1` with PowerShell logic that:
   - Iterates `.md` files in `diary/` and `sessions/`.
   - Parses lines resembling JSON (e.g. `$line.Trim().StartsWith('{')`).
   - Filters for `$obj.confidence -ge 0.9`.
   - If findings exist, calls `lean-ctx ctx_knowledge consolidate`.
   - Checks `$obj.tags` or `$obj.type` or `$obj.content` for architectural indicators (e.g. matching "architecture" or "decision") and outputs a `Write-Warning` prompting manual promotion to `AGENTS.md`.
4. Update `PROJECT.md` to mark M2 as "DONE" and M3 as the next active step.

## 5. Verification Method
- **Directories:** Run `Test-Path .lean-ctx/memory/knowledge`, `Test-Path .lean-ctx/memory/sessions`, and `Test-Path .lean-ctx/memory/diary`. All must return `True`.
- **Script:** Run `Test-Path scripts/lean-ctx-index.ps1` to confirm existence.
- **Functionality:** Create a dummy file `.lean-ctx/memory/diary/test.md` containing `{"confidence": 0.95, "type": "architecture", "content": "Use Redis for caching"}`. Run `pwsh scripts/lean-ctx-index.ps1` and verify it attempts to run the `lean-ctx` command and emits the architectural warning.
- **Config:** Inspect `.lean-ctx.toml` to ensure the memory profile settings are present.
