# MCP Tool Catalog Gateway (ModMe)

Maps lean-ctx `ctx_tools find|call` pattern to ModMe MCP servers. Terminal wrapper: `scripts/lean-ctx-tool-gateway.mjs`.

## Usage

```powershell
node scripts/lean-ctx-tool-gateway.mjs find "library docs react"
node scripts/lean-ctx-tool-gateway.mjs call context7::query-docs
```

When MCP `ctx_tools` is available in agent chat:

```
ctx_tools action=find query="create supabase migration"
ctx_tools action=call tool="supabase::list_tables" arguments={}
```

## Registered gateways

| Namespace | MCP server                      | Typical tools                          |
| --------- | ------------------------------- | -------------------------------------- |
| context7  | plugin-context7-plugin-context7 | resolve-library-id, query-docs         |
| supabase  | plugin-supabase-supabase        | list_tables, apply_migration, get_logs |
| gitlab    | plugin-gitlab-GitLab            | (project MR/issue APIs)                |

Seed source: [`scripts/collections/lean-ctx-agent-catalog.seed.json`](../../scripts/collections/lean-ctx-agent-catalog.seed.json).

## Agent catalog

```powershell
yarn agent:catalog:seed
yarn agent:catalog:validate
node scripts/lean-ctx-agent-catalog.mjs resolve --intent "observability telemetry sync"
```

Runtime catalog: `data/lean-ctx-agent-catalog.json` (gitignored; generated by seed).
