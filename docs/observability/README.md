# ModMe Observability Stack

Single entry point for all observability, telemetry, and distributed debugging tooling.

## Architecture overview

```
Collect                   Normalize (Zod)        Promote (dual-store)
──────────────────────    ─────────────────────  ───────────────────────────
session_logger JSONL  ──▶                        Supabase
agenttrace        ──────▶  telemetry-cli          ├── pipeline_runs
lean-ctx journal  ──────▶  telemetry-bridge       ├── telemetry_events
lean-ctx tee      ──────▶  @repo/schemas          ├── eval_signals
lean-ctx markers  ──────▶  observability.ts       └── trace_refs
lean-ctx archive  ──────▶
agent envelope    ──────▶                        Greptime (optional)
                                                  └── agent_spans (GREPTIME_OTEL_ENABLED=1)
```

**Correlation key:** `session_id` / `AGENT_SESSION_ID` propagated across all stores.

## Quick-start

```powershell
# 1. Load env (sets LEAN_CTX_STATE_DIR etc.)
. .\scripts\load-lean-ctx-env.ps1

# 2. Dry-run — see what would be collected
node scripts/telemetry/telemetry-cli.mjs sync --dry-run --human

# 3. Run contract tests
yarn telemetry:test:contracts

# 4. Live sync (requires Supabase secrets)
yarn telemetry:sync
```

## Commands

| Command | Purpose |
|---------|---------|
| `yarn telemetry:sync` | Full collect + normalize + promote |
| `yarn telemetry:test:contracts` | Zod golden tests + lean-ctx enum tests |
| `node scripts/telemetry/telemetry-cli.mjs sync --dry-run` | Dry-run: no writes |
| `node scripts/telemetry/telemetry-cli.mjs sync --human` | Human-readable JSON output |
| `yarn agenttrace --latest` | Latest agenttrace session summary |
| `yarn agenttrace --overview` | Cross-session cost/perf overview |

## Environment variables

| Variable | Purpose | Default |
|---------|---------|---------|
| `AGENT_SESSION_ID` | Session correlation key | Generated by `agent-session-start.ps1` |
| `LEAN_CTX_DEBUG_LOG` | Enable lean-ctx debug log | `0` (off by default) |
| `LEAN_CTX_STATE_DIR` | Journal, tee, debug logs dir | `logs/lean-ctx` |
| `LEAN_CTX_DATA_DIR` | Archive, knowledge, sessions | `data/lean-ctx` |
| `GREPTIME_OTEL_ENABLED` | Enable Greptime OTel spans | unset |
| `DEV_TENANT_ID` | Supabase tenant | `00000000-0000-4000-8000-000000000001` |
| `OBSERVABILITY_DEBUG` | Enable debug-trace in session start | `0` |

## lean-ctx collect sources (v1.1)

| Source | Event type | Path |
|--------|-----------|------|
| `lean-ctx-journal` | Human activity log | `logs/lean-ctx/journal*` |
| `lean-ctx-tee` | Shell hook failure captures | `logs/lean-ctx/tee/` |
| `lean-ctx-debug` | MCP routing trace | `logs/lean-ctx/debug.log` |
| `lean-ctx-marker` | Session adoption markers | `.cursor/hooks/state/lean-ctx-session-markers.jsonl` |
| `lean-ctx-archive` | Archive reference IDs | `data/lean-ctx/archive/` |

## Session lifecycle

```
agent-session-start.ps1
  ├── . load-lean-ctx-env.ps1
  ├── Set AGENT_SESSION_ID
  ├── Set LEAN_CTX_DEBUG_LOG=1 (if -DebugTrace or OBSERVABILITY_DEBUG=1)
  └── Write envelope.json with trace.lean_ctx paths

agent-session-finish.ps1
  ├── . load-lean-ctx-env.ps1
  ├── yarn telemetry:sync (collects lean-ctx sources)
  └── Update envelope.json with finished_at
```

## SLO targets

| SLI | Target | Source |
|-----|--------|--------|
| Session capture rate | >95% sessions have envelope | `logs/agent-orchestrator/sessions/` |
