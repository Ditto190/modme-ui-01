## Quick start (ModMe ports)

From the **Monorepo_ModMe** repository root:

```bash
yarn dev:forge:docs        # Mintlify ŌåÆ http://localhost:3104
yarn dev:forge:storybook   # Storybook ŌåÆ http://localhost:6106
yarn dev:forge             # all next-forge apps
```

From `next-forge/`:

```bash
bun install
bun run db:push            # after setting DATABASE_URL (Neon)
bun dev --filter docs
bun dev --filter storybook
bun dev --filter app       # http://localhost:3100
```

Copy env templates:

| Template | Copy to |
|----------|---------|
| `packages/database/.env.example` | `packages/database/.env` |
| `apps/app/.env.example` | `apps/app/.env.local` (and `web`, `api`) |

## Neon database

1. Create a project at [neon.tech](https://neon.tech).
2. Copy the **pooled** connection string into `packages/database/.env`:

```bash
DATABASE_URL="postgresql://USER:PASS@HOST/DB?sslmode=require"
```

3. Push schema:

```bash
cd next-forge
bun run db:push
```

Prisma loads `.env` via `packages/database/prisma.config.ts`.

## Mintlify docs

Install the CLI globally (once):

```bash
npm i -g mintlify
```

The docs app runs on **port 3104** (avoids GenerativeUI on 3000ŌĆō3004):

```bash
cd next-forge/apps/docs
mintlify dev --port 3104
```

Or from repo root: `yarn dev:forge:docs`

## Storybook workshop

Design system workshop on **port 6106**:

```bash
yarn dev:forge:storybook
```

Browse **ModMe / Workshop** stories before migrating GenerativeUI components.

## Troubleshooting

<AccordionGroup>
  <Accordion title="db:push ŌĆö Can't reach database server">
    Replace `packages/database/.env` with your Neon pooled URL, not `localhost`, unless Postgres is running locally.
  </Accordion>
  <Accordion title="Port already in use">
    Stop GenerativeUI dev servers or use worktree port offsets (`.worktree-ports.env`).
  </Accordion>
  <Accordion title="Mintlify not found">
    Run `npm i -g mintlify` and retry `yarn dev:forge:docs`.
  </Accordion>
</AccordionGroup>
